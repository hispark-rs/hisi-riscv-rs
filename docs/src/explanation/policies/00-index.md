# HAL 政策约定

这一组文档收束 hisi-riscv-hal 的三条 API / 安全政策。它们不是单个驱动的实现说明，而是**判断一个接口能不能默认暴露、一个配置值能不能接受、一段 `unsafe` 能不能被 safe API 包起来**的共同准则。

三条政策按编号阅读：

| 编号 | 政策 | 回答的问题 |
|---|---|---|
| [01](01-typed-config.md) | 类型化配置 | 一个配置值“能写出来”是否就该“能在硅片上跑”？ |
| [02](02-stable-unstable.md) | 稳定 / 不稳定 API 门控 | 一个 `pub` API 默认暴露前，需要什么硅片证据？ |
| [03](03-safe-unsafe-policy.md) | Safe / Unsafe 政策 | 一段 `unsafe` 是否被 safe API sound 地封装？ |

这三条政策互补：类型化配置约束**输入空间**，稳定 / 不稳定门控约束**默认暴露范围**，Safe / Unsafe 政策约束**信任边界**。新增或毕业 API 时，三条都要过。
